# Criterion 1: Impact and Value to the Community

## Scoring Target

Target band: **8-10 / 10**

AlQua'a Services should show a clear, meaningful benefit to residents in a dispersed rural community by making services, opportunities, and events easier to find and act on.

## Evidence Requirements

- Define the beneficiary clearly.
- Explain the current pain point.
- Show how the prototype changes the user's outcome.
- Include measurable impact claims that can be tested during the hackathon.

## Project-Specific Examples

- A resident can discover a relevant service or event in under 3 clicks.
- A parent can filter for family or education listings without searching through multiple channels.
- A small business owner can find entrepreneur opportunities in one view.
- An older resident or family helper can use direct phone, WhatsApp, or map actions without creating an account.

## Checklist

- [ ] README names the target users.
- [ ] Community Insights document explains why the problem matters.
- [ ] Demo shows a resident finding and contacting a useful listing.
- [ ] Evidence folder includes screenshots or timing notes for the core flow.
- [ ] Claims are specific, not generic.

## Documentation Template

```md
### Impact Claim

AlQua'a Services helps [target user] by [specific benefit].

Evidence:
- [screenshot/video/test note]

Measured or observed result:
- [example: relevant listing found in 2 clicks]
```
