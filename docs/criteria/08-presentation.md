# Criterion 8: Presentation and Live Demo

## Scoring Target

Target band: **8-10 / 10**

This criterion applies only if the project reaches the final round, but the repo should prepare the team for a clear live demo.

## Evidence Requirements

- Short presentation storyline.
- Stable demo flow.
- Backup screenshots or video.
- Clear answers to feasibility and evidence questions.

## Project-Specific Examples

Live demo sequence:

1. Show the resident problem.
2. Open the app in Arabic-first mobile view.
3. Filter for a persona, such as Fatima or Mariam.
4. Find a relevant listing.
5. Open contact or map action.
6. Explain how the data model scales.

## Checklist

- [ ] Demo path can be completed in under 2 minutes.
- [ ] Presenter can explain why Challenge 4 was chosen.
- [ ] Presenter can answer where the data came from.
- [ ] Presenter can explain what is built now versus future work.
- [ ] Backup video or screenshots are ready.
