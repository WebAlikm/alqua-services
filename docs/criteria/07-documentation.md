# Criterion 7: Repo Documentation and Completeness

## Scoring Target

Target band: **5 / 5**

The repository should be understandable to a judge in two minutes and verifiable without asking the team for missing context.

## Evidence Requirements

- Complete README.
- Clear run instructions.
- Demo material.
- Evidence folder.
- Source links for research.
- Clean repository structure.

## Project-Specific Examples

- README mirrors the judging rubric.
- `docs/community-insights.md` documents demographic assumptions and source confidence.
- `docs/criteria/` shows how each criterion is being addressed.
- `evidence/` stores proof rather than burying it in chat messages.

## Checklist

- [ ] README has challenge, target demographic, solution, impact, feasibility, scalability, evidence, and run steps.
- [ ] App has a live demo URL or clear local run steps.
- [ ] Demo video link is present.
- [ ] Screenshots are committed or linked.
- [ ] No placeholder claims remain in final submission.

## Documentation Template

```md
### Verification Guide

1. Open [live URL].
2. Search for [term].
3. Filter by [audience/category].
4. Open [listing].
5. Use [contact/map action].

Expected result:
- [result]
```
