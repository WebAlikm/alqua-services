# Criterion 6: Falsifiability and Evidence

## Scoring Target

Target band: **8-10 / 10**

The project should make claims that a judge can verify from the repo, demo, and data.

## Evidence Requirements

- Specific testable claims.
- Evidence table with screenshots, tests, or demo timestamps.
- No vague impact language without proof.
- Clear note when evidence is based on proxy data.

## Project-Specific Examples

- Claim: A resident can find a relevant service in under 3 clicks.
- Claim: Search supports Arabic and English keywords.
- Claim: One data model supports service, event, and opportunity listings.
- Claim: The app can be deployed without paid infrastructure.

## Checklist

- [ ] `evidence/README.md` lists all claims.
- [ ] Test cases document expected outcomes.
- [ ] Screenshots or demo video prove the main flow.
- [ ] Community research marks proxy data clearly.
- [ ] Claims can be disproved if the app fails, which makes them falsifiable.

## Documentation Template

```md
| Claim | Evidence | Status |
| --- | --- | --- |
| [specific claim] | [file, screenshot, demo timestamp, test] | Pending / Verified |
```
