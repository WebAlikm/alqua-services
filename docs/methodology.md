# Methodology

This project uses an IB Computer Science IA-inspired workflow. The goal is not to copy the school format. The goal is to use its strongest habits: define the problem before building, justify decisions, set measurable success criteria, test the result, and evaluate with evidence.

## Why This Helps

Tatweer Hackathon round one is judged entirely from the GitHub repository. This methodology makes the repo act as the product pitch, proof, and evaluation record.

## IA-Inspired Structure

| Stage | Question | Repo Evidence |
| --- | --- | --- |
| Investigation | What problem exists, and for whom? | `docs/community-insights.md`, `docs/rationale.md` |
| Planning | What does success look like? | `docs/success-criteria.md`, `docs/timeline.md` |
| Design | Why this technical approach? | `docs/technical-justification.md`, `docs/architecture.md` |
| Development | What was built? | `src/`, `data/`, `index.html` |
| Testing | Does it meet the criteria? | `evidence/` |
| Evaluation | What worked, what failed, what next? | README validation summary |

## Development Principles

- Build the smallest complete end-to-end resident flow first.
- Prioritize mobile usability over complex features.
- Make all impact claims testable.
- Label proxy data clearly when village-level data is unavailable.
- Keep deployment realistic for a rural community setting.
- Store evidence in the repository, not only in the final presentation.

## Evaluation

The prototype will be evaluated against the success criteria in `docs/success-criteria.md`. Evidence should be placed in `evidence/`, not left in chat notes or presentation claims.

## Methodology Footnote

Data synthesized using Abu Dhabi Census reports, TDRA digital government and telecom context, UAE digital adoption metrics, Tatweer Hackathon challenge material, and regional community news coverage of the Al Ain desert region. Village-level Al Qua'a demographic figures were not found in public official sources, so this repo labels regional or national proxy data clearly.

## Source Notes

- SCAD is used for Abu Dhabi and Al Ain regional census context.
- DataReportal is used for UAE national digital adoption context.
- TDRA is used as official UAE telecom and digital government context, not as village-level evidence.
- Tatweer Hackathon materials are used as the authoritative source for the challenge problem statement.
