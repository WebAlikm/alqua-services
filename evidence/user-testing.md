# User Testing Evidence

Evidence for SC3: Quick discovery - User finds relevant listing in ≤3 clicks

**Date:** June 27, 2026  
**Platform:** Al Qua'a Services  
**URL:** https://alqua-services.vercel.app  
**Method:** Task-based usability testing  
**Participants:** 3 (minimum for lightweight validation)

---

## Test Overview

### Success Criteria Being Validated

**SC3:** Quick discovery - User finds a relevant listing in 3 clicks or fewer

**Additional metrics tracked:**
- Task completion rate (target: 80%+)
- Time to completion (target: <60 seconds)
- Ease of use rating (target: ≥4/5)
- User feedback (qualitative)

---

## Test Protocol

### Standard Test Procedure

**Duration:** 5-10 minutes per participant  
**Setting:** Remote or in-person  
**Device:** Participant's own device (realistic usage scenario)

**Script:**

1. **Introduction (1 min):**
   > "Thank you for testing Al Qua'a Services. This is a prototype for 
   > finding local services. I'll give you a task - just try to complete 
   > it naturally. There are no wrong answers. If something confuses you, 
   > that's valuable feedback."

2. **Task Assignment (30 sec):**
   > "Imagine you [need scenario]. Use this website to find it."

3. **Observation (2-5 min):**
   - Don't help unless completely stuck
   - Count clicks
   - Time the task
   - Note confusion points

4. **Debrief (2 min):**
   - "How easy was that? (1-5 scale)"
   - "What was confusing?"
   - "Would you use this in real life?"
